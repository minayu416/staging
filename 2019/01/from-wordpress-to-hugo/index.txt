<h2>找尋資料</h2>
<p>對於Hugo還一無所知，Hugo與Wordpress這種CMS是不太一樣的系統

我前後參考了幾了網站來了解Hugo是一個 靜態網站生產器

一開始覺得很不靠鋪，我也是wordpress用戶，少了資料庫的靜態網頁生產器怎麼能媲美功能美滿的CMS呢?

後來我錯了, 架好Hugo後，我發現我可以離開Wordpress的限制，自由的改動網站的外觀，以及添加任何我想要的功能</p>
<p>非常的自由，但要有一個認知就是</p>
<p>所以我看了些比對的文章，像是選個最好的靜態網頁生產器, Jekyll vs Hugo

首先也要記得參觀Hugo官網,選主題時你會常常在主題頁面中停留

用Hugo怎能不看Hugo官網呢?

再來我擔心沒有後台我就變成了不會寫文章的手殘

所以我找到了跟Hugo與Github相容的CMS: Forestry

(可是我昨天實際用過Forestry，不可否認很好用，但我現在卻正在調查她是不是造成我的網頁掛掉的兇手 ...，等我查明後才會推薦，這讓我的心情很鬱悶)

有機會分下個文章跟大家分享</p>
<h2>架設Hugo</h2>
<p>我找了一堆網站，來輔助我架設Hugo</p>
<p>等到我真的開始實作後，我發現完全架不起來</p>
<p>這份簡報是我歷經千辛萬苦，架了幾十次，跟著架才成功的指南</p>
<p><code>讚</code></p>
<p><strong><a href="https://speakerdeck.com/fukuball/create-static-website-with-hugo-in-2-minutes">請服用</a></strong></p>
<p> </p>
<p><code>我是使用Mac</code></p>
<h3>首先安裝Hugo</h3>
<pre><code>$ brew install hugo
</code></pre>
<p>如果沒有brew，可以試試看
<em><strong><a href="http://osxdaily.com/2018/03/07/how-install-homebrew-mac-os/">這篇文章</a></strong></em> 裝一下</p>
<p>＊註： <code>brew</code> 就是mac系統的套件管理系統 <code>Homebrew</code>, 若是沒有 homebrew，可以找尋安裝指南，幾行指令就搞定!</p>
<h3>新建一個網站</h3>
<pre><code>hugo new site mysite
</code></pre>
<p>此時您的網站已經初始生成</p>
<h3>進入網站資料夾</h3>
<pre><code>cd mysite
</code></pre>
<p>內部的資料夾包刮 content, theme, layout, ....</p>
<h2>安裝Theme</h2>
<p>接著就去Hugo官網挑挑你愛的主題吧!

我是挑 <em><strong><a href="https://themes.gohugo.io/hugo-tranquilpeak-theme/">Tranquilpeak</a></strong></em>

每個主題點進Download都會進到該主題的Github

那我們就用Git指令把主題裝到我們的網站資料夾內</p>
<p>進到主題資料夾安裝</p>
<pre><code>$ cd themes
</code></pre>
<p>這邊以Tranquilpeak為例</p>
<pre><code>$ git clone https://github.com/kakawait/hugo-tranquilpeak-theme.git
</code></pre>
<p>裝好後利用ls指令會看到hugo-Tranquilpeak-theme資料夾在您眼前</p>
<pre><code>$ ls
hugo-Tranquilpeak-theme
</code></pre>
<p>接著看你要使用編譯器，還是手動拉</p>
<p>將一些相關文件從theme複製到你的根目錄</p>
<p><em><strong>將你的主題theme資料夾內 "archetyps" 內的檔案複製到你的"archetyps"</strong></em></p>
<p>這邊是用hugo-Tranquilpeak-theme 主題</p>
<p>將<code>hugo-Tranquilpeak-theme/archetyps/</code>內容移動至 <code>mysite/archetyps/</code>內</p>
<p><em><strong>將你的主題theme資料夾內 exampleSite 內的檔案, 直接取代根目錄的檔案</strong></em></p>
<p>忘記在哪邊看到，在簡報中是沒有看到這個部分</p>
<p>要順利先將主題的頁面刻出來，就將 <code>exampleSite</code> 內像是content, static 都直接複製取代掉你的<code>mysite/</code> 中的content 及 static</p>
<p><em><strong>將你的主題theme資料夾內 exampleSite 內的config.toml, 直接取代掉根目錄的config.toml</strong></em></p>
<p>就是取代<code>mysite/config.toml</code> 中</p>
<p>接下來回到根目錄</p>
<p>將hugo run 起來 就能看到完整的主題頁面</p>
<pre><code>$ hugo server
</code></pre>
<p>先求有再求好</p>
<p>接下來就是您的工作囉~</p>
<p>打開根目錄的config.toml 檔案, 將參數改一改，改成你要的</p>
<h2>將Wordpress文章輸出, 轉為，並放入Hugo</h2>
<p>當初要實作時，我參考了一個網站，發現完全不work</p>
<p>我也實際參觀Hugo migrations的頁面，照著上面的一些功能，也是不work xD</p>
<p>後來參照from wordpre to hugo 的文章後，使用了Exitwp ，才順利把文章從xml 轉到md檔</p>
<p> </p>
<h3>Wordpress後台 Export 你的文章 .xml</h3>
<pre><code>[你的wordpress url]/wp-admin/
</code></pre>
<p>可以進到wordpress的管理員後台</p>
<pre><code class="language-">點選 Tools > Export
</code></pre>
<p>可以進去Export文章的網頁</p>
<p>＊若沒有Export 要去 Plugin 安裝，wordpress.com都會有，Wordpress.org可能要安裝套件Plugin</p>
<pre><code>選擇免費的 Start Export
</code></pre>
<p>開始把你的部落格點匯出，完成的檔案會寄信到您的信箱</p>
<p> </p>
<h3>用外部套件將Wordpress XML 轉為Hugo 使用md</h3>
<p>參考網站: from wordpre to hugo,  Exitwp</p>
<p> </p>
<p>將 exitwp 下載下來</p>
<pre><code>git clone https://github.com/thomasf/exitwp.git
</code></pre>
<p> </p>
<p> </p>
<p>＊請注意這個套件的環境為python 2.7</p>
<pre><code>pip install pip_requirements.txt
</code></pre>
<p>或者</p>
<pre><code>pip install
</code></pre>
<p>驅動python程式來轉檔</p>
<pre><code>python exitwp.py
</code></pre>
<p>幸運沒報錯的話，就可以進<code>build</code>資料夾領取md囉~</p>
<p> </p>
<h3>將轉出的.md檔放進hugo內</h3>
<p>這個部分老實說要看不同的主題會有不同的要求，主要看你的 網站資料夾 內的 "archetyps" / themes/你的主題/archetyps/內的.md檔怎麼定義一篇部落格</p>
<p>將輸出的.md檔放進Hugo, 通常會是這幾個資料夾，取決於你的theme/exampleSite/content 內容</p>
<pre><code>content/blog
content/post
content/posts
</code></pre>
<p>確認一下每篇md檔有沒有和archetyps 內.md規範符合</p>
<p>至少要有archetyps 內.md 內規定的幾個元素</p>
<p>有可能是</p>
<pre><code>title
layout
</code></pre>
<p> </p>
<p>或</p>
<p>最後，到根目錄跑一次建置，進入 1313 看一下網頁有沒有問題，就可以準備部署囉!</p>
<pre><code>hugo server
</code></pre>
<h2>部屬至Github</h2>
<p>將網頁建在哪都可以，AWS, GCP 或其他平台，而我看大部分的人都部屬在github上</p>
<p>這部分我自己做的也不是很好，沒辦法建議甚麼qq</p>
<p>2019.05.19</p>
<p><em><strong>跟大家說個福音</strong></em>，那就是, 隨著我後來的功力大增，我終於知道怎麼好好的部署至Github啦！！！！</p>
<p>傳送門在此: <em><strong><a href="/2019/04/how-to-host-hugo-on-github/">[BLOG] 如何在GITHUB上架設HUGO網站</a></strong></em></p>
<p>當然還是可以繼續往下讀，只是當時不怎麼會弄寫得不怎麼好就是XD</p>
<p>你可以參考hugo本身的網站</p>
<p> </p>
<p>但我用的方法:<a href="https://medium.com/@chs_wei/%E5%9C%A8-github-%E9%83%A8%E7%BD%B2-hugo-%E9%9D%9C%E6%85%8B%E7%B6%B2%E7%AB%99-9c40682dfe40">這位部落格主提供的方法</a></p>
<p>在github分成兩個repo，一個是放你build網站的程式資料夾(當然你也可以選擇不存)</p>
<p>另外一個則是取名為</p>
<p> </p>
<p>這個指令會生成public資料夾</p>
<p> </p>
<p>接著再按照新開的repo上有指南，把git init 後，往github repo 推送</p>
<p> </p>
<p>應該不用幾分後，進入你的網站</p>
<p>就可以看到你的網站囉!</p>
<p> </p>
<p>除了注意一下環境要是python2.7外，若想使用python3執行</p>
<p>我想可以試試看在exitwp.py 程式裡，把print 換成 print() 語法</p>
<p>我有試過，但我忘了有沒有成功</p>
<p> </p>
<p>＊exitwp系統說缺少yaml，可是我執行 pip install yaml 卻說沒有這個套件?</p>
<p>正確的套件是 PyYAML</p>
<p> </p>
<p>＊theme 內只有theme.toml 沒有 config.toml?</p>
<p>theme 內的 config.toml 在 範例網站的資料夾內</p>
<p>應該會有個 exampleSite 的demo 網站\，裡面有demo用的config.toml</p>
<p> </p>
<p>＊為什麼我hugo server後畫面是空白的?</p>
<p>很多教學文章都是說將theme clone下來後，把config.toml檔改一改就好了</p>
<p>但實際上需要把 exampleSite 內的 archetyps, static 及 content 最好都取代根目錄的資料</p>
<p>(注意不要把exampleSite 內的content/post內容取代掉你的文章)</p>
<p>確認主題可以顯示後，再把你的部落格文章.md檔放進 content內</p>
<p> </p>
<p>＊為什麼我把我的文章放進content後跑不出來？</p>
<p>我想這要參考一下 theme 內的 archetyps 怎麼定義為一篇文章</p>
<p> </p>
<p> </p>
<p>以上大致上是我架站時發生的一些問題</p>
